Recommended tech stack

Optimised for a fast, maintainable, single-developer build that the Integreon team can ship incrementally. Every choice prioritises developer velocity and UI performance.

Frontend React 18 + Vite

Why: Vite's HMR makes iteration instant. React's component model maps perfectly to Leah's repeating patterns (field groups, user rows, contract cards). No build config overhead.

react 18vite 5react-router-dom
UI Components shadcn/ui + Tailwind

Why: shadcn gives you copy-paste composable components (Table, Dialog, Select, Command palette) — no library lock-in. Tailwind makes layout changes instant without leaving JSX.

shadcn/uitailwindcsslucide-react
Data fetching TanStack Query v5

Why: Automatic caching, background refetch, and stale-while-revalidate make the app feel instant. Mutations with optimistic updates. Handles Leah's paginated lists perfectly with useInfiniteQuery.

@tanstack/react-queryaxios
State Zustand

Why: Lightweight global store for auth token, selected tenant, config. No boilerplate compared to Redux. Persists token to localStorage automatically with a plugin.

zustandzustand/middleware persist
Tables TanStack Table v8

Why: Headless table engine that powers all 11 data grids (users, contracts, clauses, fields). Column sorting, filtering, virtual scrolling for 1000+ row datasets — all without UI library lock-in.

@tanstack/react-table@tanstack/virtual
Forms React Hook Form + Zod

Why: Zero re-renders on every keystroke (unlike Formik). Zod gives runtime validation that exactly mirrors Leah's API payload requirements — catch payload errors before the API call.

react-hook-formzod@hookform/resolvers
CORS proxy Vite dev proxy / Node.js

Why: Leah APIs don't have CORS headers for browser requests. Vite's built-in proxy forwards calls in dev. For production, a minimal Express.js reverse proxy handles auth header injection.

vite proxy configexpresshttp-proxy-middleware
Export xlsx + jsPDF

Why: Bulk import templates, user reports, audit exports all need Excel. jsPDF handles certification reports and test results. Both run entirely in the browser — no server needed.

xlsx (sheetjs)jspdfjspdf-autotable

Why this stack makes the app feel fast

  • TanStack Query cache

    App types, users, field metadata load once and are reused — navigation feels instant.

  • Virtual scrolling

    400 users or 1000 contracts render in <16ms because only visible rows are in the DOM.

  • Optimistic updates

    Unmasking a user or editing a field feels instant — UI updates before the API responds.

  • Zero-config CORS in dev

    Vite proxy means no browser extension or backend needed during development.
